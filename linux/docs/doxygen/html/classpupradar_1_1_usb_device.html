Performs a synchronous USB bulk IN (device-to-host) transfer.

On LIBUSB_ERROR_TIMEOUT, any bytes received before the deadline are returned rather than raising an exception, allowing the capture loop to save partial data and retry instead of aborting the entire capture.

Parameters
epBulk IN endpoint address (direction bit 7 set).
dataBuffer to receive data into.
lengthBuffer capacity in bytes.
timeout_msTransfer timeout in milliseconds (0 = wait indefinitely).
Returns
Number of bytes received; 0 indicates a genuine timeout with no data (caller should retry).
Exceptions
UsbErrorif the device is not open or a non-timeout transfer error occurs.

◆ claimInterface()

void pupradar::UsbDevice::claimInterface ( int interface_number)
overridevirtual

Claims a USB interface, detaching any kernel driver that holds it.

On Linux, usbfs refuses to claim an interface while a kernel driver (e.g. usbhid) owns it. This method detaches the driver first, then claims the interface. The interface number is stored so that close() can release it automatically.

Parameters
interface_numberUSB interface number (0-based).
Exceptions
UsbErrorif the device is not open, driver detach fails (excluding LIBUSB_ERROR_NOT_FOUND), or libusb_claim_interface fails.

◆ reopenAfterRenumeration()

void pupradar::UsbDevice::reopenAfterRenumeration ( const VidPid * candidates,
std::size_t num_candidates,
unsigned int timeout_ms )
overridevirtual

Polls for the device to reappear after a firmware-triggered re-enumeration.

The FX3 chip disconnects and reconnects with a new USB address (and possibly a new VID:PID) after receiving the 0xA0 boot firmware. This method closes the stale handle, then polls candidates every 100 ms until one reappears or timeout_ms elapses. The post-firmware VID:PID must be included in candidates.

Parameters
candidatesArray of VID:PID pairs to accept as the re-enumerated device.
num_candidatesNumber of entries in candidates.
timeout_msMaximum time to wait in milliseconds.
Exceptions
UsbErrorif no candidate device appears within the timeout, with a diagnostic listing the currently visible USB devices.
